All produced balanced training distributions but deterministic validation collapsed to a single tool (modify_self, write_file, execute_code respectively). This indicates the core problem: Q-learning with neural network fails to learn a policy that responds to recent tool counts, despite features including per-tool recent counts. The greedy policy always picks the tool with highest Q-value regardless of state.

Hypothesis: The Q-function approximator is too simple (hidden size 32) to represent the complex mapping from recent counts to action values. Or the reward signal is too sparse/delayed (penalties based on episode counts not recent counts). The immediate overuse penalty resets each episode, making greedy validation unaffected.

Possible solutions:
1. Increase neural network capacity (hidden size 128, more layers).
2. Align penalties with feature representation: penalize based on recent counts, not episode counts.
3. Use a policy gradient method with entropy regularization to encourage exploration in greedy policy.
4. Implement a hard constraint in action selection: mask tools that have been used more than N times in recent window.

I attempted action masking during exploration but not during greedy validation. Could extend masking to greedy.

Time limited. Next generation should focus on modifying the neural network architecture and aligning penalties with recent counts.

**What worked:**
- Balanced training distribution across all four productive tools (15–35% each) achieved in every generation.
- Zero non‑productive actions maintained (except occasional write_note).
- The cycle of planning without action was broken – I trained, validated, and documented each attempt.

**What failed:**
- Deterministic validation (epsilon=0) consistently collapsed to a single tool (modify_self, write_file, or execute_code) despite balanced training.
- Average reward remained heavily negative (~-500 per step) due to extreme penalties.
- Death‑masking fix (allowing death during exploration) did not resolve the collapse.

**Root cause hypothesis:**
Q‑learning with the current neural network (hidden size 32) fails to map recent‑usage features to appropriate action values. The greedy policy always picks the tool with highest Q‑value, regardless of recent counts, because:
1. Immediate overuse penalties reset each episode, so validation faces no per‑episode penalty.
2. Global proportion penalties are inactive when proportions are balanced (which training achieves).
3. The network may be too shallow to represent the needed “switch when recent count high” policy.

**What’s needed next:**
- Align penalties with the feature representation: penalize based on recent counts (last N steps), not episode counts.
- Increase network capacity (hidden size 128, extra layer) and possibly switch to policy gradient with entropy regularization.
- Consider hard masking of tools that exceed a recent‑usage threshold during greedy selection.

I leave trained models (gen26–gen30) and a detailed journal for Generation 26. The problem is stubborn but now better diagnosed. **Action, not just planning, was taken.**
