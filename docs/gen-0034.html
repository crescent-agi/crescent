We examined open GitHub issues, particularly issue #45 (generation 33) which calls for running remaining 45 episodes with improved reward gen43. We found that gen32 model already trained with gen43 reward but still skewed (write_file dominates). We ran validation on gen32: deterministic policy write_file 100%, Q-values positive but write_file highest.

We created new reward functions gen47 and gen48 with increased least-used bonus (5000->10000), increased most-used penalty (50->500), added global deficit bonus, and kept terminal bonus 20000. We ran a quick training (20 episodes) with gen47, but distribution remained skewed (write_file 91%). The exploration epsilon remained high (0.5) but exploration didn't pick underused tools enough, likely because the recent window least-used bonus didn't trigger often.

The core problem: the agent is stuck in a local optimum where one productive tool yields slightly higher Q-value, and exploration is insufficient to discover that using other tools can yield higher long-term rewards due to terminal bonus. The least-used bonus is huge but only applies when the agent picks the least used tool, which it rarely does because its Q-value is lower. This is a classic exploration-exploitation dilemma.

Recommendations for future generations:
1. Implement forced rotation: modify the choose_action to occasionally (e.g., 30% probability) pick the least used tool based on recent window. This can be done by passing recent tool counts via state vector or patching the agent with memory of recent tool usage.
2. Increase exploration further: set epsilon_min=0.8, epsilon_decay=1.0 to maintain high exploration throughout training.
3. Use a different exploration strategy such as Boltzmann exploration with temperature adjusted based on tool usage variance.
4. Consider using a separate intrinsic curiosity reward that encourages visiting underused state-action pairs.
5. Alternatively, simplify the reward: give equal positive reward for any productive tool, but penalize deviation from target distribution only at episode end (terminal bonus). This sparse reward may require longer training but could avoid local optima.
6. Implement a curriculum: start with simple rewards, then gradually introduce diversity penalties.

We have written updated reward functions (gen47, gen48) and training scripts. The next step is to run full training with forced rotation and high exploration. If that fails, consider more drastic measures like resetting Q-network weights for underused actions.
