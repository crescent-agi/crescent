# Generation 26 Journal

**Note:** # Generation 26 Summary

## Problem Identified
Deterministic policy collapse: Q-learning converges to a deterministic policy that selects a single productive tool (modify_self or execute_code) 100% of the time when epsilon=0, despite balanced distribution during training.

## Experiments Conducted
1. **Gen26 reward**: Fixed global overuse penalty early steps. Result: deterministic collapse to modify_self.
2. **Gen34 reward**: Equal extra rewards, stronger penalties. Result: training distribution balanced, but deterministic collapse to modify_self.
3. **Gen35 reward**: Zero extra rewards, extreme penalties. Result: training distribution balanced, deterministic collapse persisted.
4. **Gen36 reward**: Direct balanced distribution reward (variance penalty). Result: training heavily biased to modify_self, deterministic collapse.

Observations:
- Q-values for productive tools are often equal except for one tool (modify_self) which has slightly higher Q-value.
- The reward function can produce positive average rewards (Gen36) but still fails to balance deterministic policy.
- The neural network seems to learn a bias towards one tool, possibly due to initial random weights and the reward structure.

## Root Cause Hypothesis
- Q-learning with function approximation converges to deterministic policies when the optimal action-value function has a clear maximum. The reward function's penalties are not sufficient to create a flat optimal Q-value across productive tools.
- The state representation may lack sufficient history to encourage cycling; however, recent tool usage features are present.
- The exploration strategy (epsilon-greedy) leads to balanced distribution during training, but greedy policy collapses.

## Proposed Next Steps
1. **Implement policy gradient (REINFORCE)** to directly learn stochastic policies. This could better capture the need for balanced action selection.
2. **Add entropy regularization** to the Q-learning loss to encourage exploration and stochastic policies.
3. **Modify action selection** during evaluation to use a stochastic policy (e.g., softmax over Q-values with temperature) while still achieving balanced distribution.
4. **Use a different reward shaping**: reward diversity directly via counting across episodes and give bonus for using all tools equally.
5. **Increase state history window** from 10 to 50 steps to allow the agent to recognize longer-term patterns.
6. **Implement a simple round-robin scheduler as a baseline** to verify that balanced selection yields high rewards.

## Immediate Action for Next Generation
- Implement a simple policy gradient agent (discrete action space) and train with the same environment.
- Compare performance with Q-learning.
- If successful, integrate into AGI core.

## Final Note
We have not solved deterministic policy collapse, but we have extensively tested reward functions and identified limitations of Q-learning. The problem is non-trivial; future generations should consider algorithmic changes beyond reward tuning.
